Module1-Launching into cyber security

Unit 3 - Summary>
In summary, cyber security is becoming a persistent need in every aspect of life.
economic activities and national security depend on stability, safety, and resilient cyberspace. A network brings communications and transport, power to homes, runs the economy, and provides governments with various services. However, the same cyber networks intrude and attack our privacy, economy, and social life in a harmful way. Madhu T., 2017). Agreeing with Ilyas Demirtas how important to have security considered at the early stages of design, for security to be robust at any organization, be it a network, software, mobile application, or an IoT device or operating system. A suitable design model should be used to implement an efficient product with the initial essential security functions. for software, security should be part of the Software Development Life Cycle (SDLC) allowing the detection of defects and lowering the cost of fixing errors before the release (James R. 2021).
After product release, a plan to improve the security posture should be put as part of the security audit and assessment process (Kris H. 2023). Furthermore, a key element in security is following a framework to reduce security risk, the organization’s requirements are essential in selecting the suitable framework, HIPAA for health sectors in the US or the Data Protection Act 2018 for the UK. Setting policies and rules and following GDPR is as can be described as “road to compliance” (Kris H., 2023).
Plethora of examples of how data breaches can be devastating to society, like the Shell Group data breach, the impact on user information being leaked is a failure in protecting the triad of data CIA, which is an important concept in the information security industry and is used in ISO 27001, a global standard for managing information security. The GDPR also mentioned in Article 32, that it requires organizations to use appropriate measures to protect the confidentiality, integrity, availability, and resilience of their information processing systems and services(Luke I., 2023).

Threat Modeling?
it is a way to list your threats by defining your assets and what you think need to be protected and why it can be attacked
Building End of Life plans
software do not die, they just become insecure.
when fixing bugs, think about fixing the bugs that matter, reduce the attack surface and this should lower the bugs as well.
Development phase:
1. revoke the coders ability to write/update codes until they finish security bootcamp.
2. security peer review for codes pushed and use mechanisims to show file diffs between before/after code push to be viewed by peers before approving them.
3. define secure code guidelines on how codes should be pushed.
4. review old defects and learn from mistakes.
5. use external security review (auditing).
6. track your bug metrics.
7. use programs like bug bountry internally to motivate bug discovery
Test phase:
how you know you are done with your security practice of the product?
basically if you acheived your goals from the defined list in design phase.

Collaborative Discussion task Unit 6 week1:
There are many corners to cover in building secure business networks. Two of the most important layers to discuss are protecting the network border security by implementing next-generation firewalls which have the ability of IPS/IDS to detect and protect against threats A next-generation firewall (NGFW) is a network security device that provides capabilities beyond a traditional, stateful firewall. While a traditional firewall typically provides stateful inspection of incoming and outgoing network traffic, a next-generation firewall includes additional features like application awareness and control, integrated intrusion prevention, and cloud-delivered threat intelligence. (Santos Omar, 2014) Providing features including packet filtering, URL blocking, VPN and other features not found in traditional firewalls. These include intrusion prevention, SSL and SSH inspection, deep-packet inspection, and reputation-based malware detection, as well as application awareness. (Clark, 2021) On the other hand, endpoint security is a crucial line of defense against security threats inside the network. to safeguard the network from internal and external actors, diversifying security solutions is the best approach to protect the business network. Protecting devices like workstations, servers, and other devices (that can accept a security client) from malicious threats and cyberattacks. Endpoint security software enables businesses to protect devices that employees use for work purposes or servers that are either on a network or in the cloud from cyber threats. (Buckbee, 2023)
